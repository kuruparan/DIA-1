Configure it
User to Server: all sms should be sent to 77100

Subscribe: REG dia
UnSubscribe: UNREG dia

configure(USSD) : *771*100#

on: dia on
off: dia off

if more than one devices configured

on: dia on devicename
off: dia off devicename

schedule: dia 1110101:4.00-6.00:12.30-1.00:17.00-19.45 [devicename]
We recommend you to schedule using web interface :)
The message schedules per week first 7 bits says from Monday to Sunday which days the schedule should work and not by indicating 0 and 1. after that : separated on-off times in 24Hrs, device name is optional if user is controlling more than one device

modes of operation

1. Default mode: dia mod d
Device just follow the schedule and commands
2. Alert mode: dia mod a
Device just follow the schedule and commands and sends alerts+suggestions about the environment on critical situations.
3. Intelligent mode: dia mod i
Device uses sensors and decide how to water the plant efficiently, only ask the user if can’t make decision.

Reporting
1. Default: dia rep all
Send status sms whenevers a command/intelligent action executed.
2. Summary: dia rep diay send a summary sms per diay dia rep week send a summary sms per week
3. No reports: dia rep off No status sms send back.
